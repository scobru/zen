## 1. Vấn đề cốt lõi

## 2. Quyết định kiến trúc quan trọng

### 2.1 Wire format của `/status`

`/status` **không trả JSON**. Trả về **compact signed string** của ZEN:

```
<86 base62 sig><recovery bit 0|1>:<payload JSON>
```

Client:
```js
const str = await fetch(base + "/status").then(r => r.text())
const pub = await ZEN.recover(str)           // pub từ chữ ký — không tự khai
const data = await ZEN.verify(str, pub)      // xác minh và lấy payload string
const status = JSON.parse(data)              // { name, version, domain, peers, ... }
```

Ưu điểm:
- `pub` không tự khai — được suy ra từ chữ ký (không thể giả mạo)
- Không cần `signed` field riêng trong JSON
- Nhất quán với triết lý ZEN: truth from crypto

### 2.2 Chỉ có một route: `/status`

`/peers` **bị xóa**, không giữ alias.

Tuy nhiên: **AXE (`lib/axe.js`) hiện tại gọi `/peers` và expect JSON array**. Đây là browser bundle code (`zen.js`). Phải cập nhật AXE để:
1. Gọi `/status` thay vì `/peers`
2. Recover pub, verify, parse JSON, lấy `status.peers` array
3. Gain bonus: biết `pub` của relay ngay lúc connect

```js
// lib/axe.js — thay đoạn fetch /peers:
fetch(base + "/status", ...)
  .then(r => r.ok ? r.text() : null)
  .then(async function(str) {
    if (!str) return;
    try {
      var pub = await ZEN.recover(str);
      var data = await ZEN.verify(str, pub);
      var status = JSON.parse(data);
      if (Array.isArray(status.peers)) status.peers.forEach(adpbr);
      // TODO future: store pub → relay identity cache
    } catch {}
  })
```

**Hệ quả:** Sau khi build `zen.js`, tất cả browser client dùng ZEN cũ (`/peers`) sẽ không nhận được peer list. Phải cân nhắc migration window.

**Quyết định cần xác nhận:** Có cần keep `/peers` alias trong thời gian chuyển đổi không?

### 2.3 Graph soul: `~pub/status` (đổi từ `~pub/mcp/info`)

MCP server hiện publish:
```js
zen.get("~" + self.pub + "/mcp/info").put({
  name: "zen-mcp", version, pub, relay: true
}, null, { authenticator: self })
```

Đổi thành `~pub/status`. Format soul thống nhất với HTTP route `/status`.

Về **nội dung** soul: khác với HTTP `/status`, soul được ZEN auth (`authenticator: self`) bảo vệ sẵn — không cần signed string bên trong. Lưu plain object là đủ, vì ZEN's HAM + auth đã ký.

```js
zen.get("~" + self.pub + "/status").put({
  name: "zen",
  version: packageVersion,
  domain: domain || null,
  pub: self.pub,
  mcp: true,       // đây là MCP server
  relay: true,     // backward compat
  peers: [...kprs] // các peers biết tới
}, null, { authenticator: self })
```

HTTP relay (server.js) không publish vào graph — nó trả `/status` qua HTTP. MCP server publish vào graph để agent có thể subscribe mà không cần HTTP.

### 2.4 Nguồn IP — Một nguồn duy nhất, không drift

Hiện trạng:
- `server.js` gọi `disc({ noSave: true })` 1 lần (async, fire-and-forget) để lấy `ip6` → thêm `surl6` vào `kprs`
- `disc()` trả `{ domain, ip, ip6, port, source }` — có cả `ip` (IPv4)
- Kết quả **không được lưu** vào biến — bị mất sau khi dùng xong

**Thay đổi:** Lưu kết quả `disc()` vào `let discResult = null`. Dùng cho:
1. `/status` response (ip4, ip6, domain)
2. Periodic refresh (IP thay đổi)

```js
let discResult = null;

async function refreshDisc() {
  try {
    discResult = await disc({ noSave: true });
    if (discResult.ip6) {
      const scheme = opt.key ? "wss" : "ws";
      surl6 = scheme + "://[" + discResult.ip6 + "]:" + port + "/zen";
      kprs.add(surl6);
    }
  } catch {}
}

refreshDisc(); // gọi ngay khi start
setInterval(refreshDisc, 10 * 60 * 1000); // refresh mỗi 10 phút
```

MCP server cũng cần IP — nhưng MCP server import từ server context hay chạy độc lập?  
→ Hiện tại MCP không có IP. Chưa cần — MCP agent kết nối qua pub key, không cần IP trực tiếp.  
→ Không add IP vào MCP bây giờ để tránh drift.

### 2.5 `peers` trong `/status` — kế thừa RTT sort

Giống hệt `/peers` cũ: dùng `rttOf()` để sort trước khi đưa vào payload.

```js
const sortedPeers = [...kprs].sort((a, b) => rttOf(a) - rttOf(b));
```

## 4. Design của `lib/status.js`

```js
// lib/status.js — Shared status payload builder
import { createRequire } from "node:module";
const require = createRequire(import.meta.url);
const { version } = require("./package.json");
import ZEN from "../index.js";

// buildStatus(fields) → plain object (no signed field)
// fields: { pub, domain, ip4, ip6, port, peers, mcp }
export function buildStatus(fields) {
  return {
    name: "zen",
    version,
    domain: fields.domain ?? null,
    pub: fields.pub ?? null,
    mcp: fields.mcp ?? false,
    peers: fields.peers ?? [],
    timestamp: Date.now(),
    ip4: fields.ip4 ?? null,
    ip6: fields.ip6 ?? null,
    port: fields.port ?? null,
  };
}

// signStatus(payload, pair) → compact signed string
// Returns ZEN.sign(JSON.stringify(payload), pair)
export async function signStatus(payload, pair) {
  return ZEN.sign(JSON.stringify(payload), pair);
}
```

HTTP `/status` handler:
```js
if (req.method === "GET" && (req.url === "/status" || req.url === "/status/")) {
  res.writeHead(200, {
    "Content-Type": "text/plain; charset=utf-8",
    "Access-Control-Allow-Origin": "*",
  });
  if (!identity) { res.end(""); return; }  // no identity = no signed status
  const payload = buildStatus({
    pub: identity.pair.pub,
    domain,
    ip4: discResult?.ip ?? null,
    ip6: discResult?.ip6 ?? null,
    port,
    peers: [...kprs].sort((a, b) => rttOf(a) - rttOf(b)),
    mcp: false,
  });
  signStatus(payload, identity.pair).then(str => res.end(str)).catch(() => res.end(""));
  return;
}
```

## 7. Migration concern — AXE `/peers` → `/status`

AXE là browser code, được bundle vào `zen.js`. Khi chúng ta đổi:
- **Relay mới** (với `/status`) + **browser dùng zen.js cũ** → browser gọi `/peers` → 404 → không nhận được peers
- **Relay mới** + **browser dùng zen.js mới** → OK

Không thể migration window im lặng vì là breaking change cho browser clients.

**Hai lựa chọn:**

**Option A — Hard cut (recommended, user's intent):**  
Xóa `/peers`, chỉ có `/status`. Build + publish `zen.js` mới. Browser clients phải upgrade. Phù hợp vì đây là internal project.

**Option B — Transient alias (30 ngày):**  
`/peers` vẫn hoạt động nhưng redirect hoặc alias sang `status.peers`. Xóa sau khi confirm không còn client cũ.

→ **Cần xác nhận từ user**: Option A hay B?
