## 2. Phân tích MCP bridge timeout

### Triệu chứng

Tất cả MCP tools (`zen-push`, `zen-identity`, `zen-status`) đều trả về:
```
McpError: MCP error -32001: Request timed out
```

### Nguyên nhân

MCP bridge process (PID 10336, khởi động ngày 15/05):

```
x 10336 node /home/x/.npm/_npx/.../node_modules/.bin/mcp mcp
```

**Timeline:**
1. MCP bridge khởi động 15/05, kết nối tới IPC socket `/home/x/.local/share/zen/mcp.sock`
2. `zen.service` restart ngày 17/05 11:47 → socket cũ bị xóa, socket mới được tạo
3. Bridge code cũ: khi socket close, gọi `recover()` → `standalone()` → tạo ZEN peer mới
4. Standalone ZEN peer này không connect được hoặc bị timeout
5. **17 giờ sau** — mọi MCP call vẫn hang forever

**Test trực tiếp qua socket mới** (bypass bridge):

```bash
echo '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"push","arguments":{"to":"0wTwUp...","data":{"hello":"test"}}}}' \
  | nc -U /home/x/.local/share/zen/mcp.sock -w 5

# Kết quả: {"jsonrpc":"2.0","id":2,"result":{"content":[{"type":"text","text":"{\n  \"ok\": true\n}"}]}}
```

→ **`zen.push` hoạt động bình thường**. Vấn đề chỉ ở bridge process cũ bị stale.

## 3. Fix MCP bridge — reconnect loop

**Commit:** `64beb61` — `fix: mcp bridge reconnects to IPC socket after zen.service restart`

### Vấn đề cũ

```js
// lib/mcp.js — code cũ
function recover() {
  socket.destroy();
  standalone();  // ← spawns isolated ZEN peer, often fails or hangs
}
socket.once("close", () => { if (bridged) recover(); });
```

Khi service restart:
1. Socket close → `recover()` → `standalone()` ← sai hướng
2. `start()` tạo ZEN peer mới mất vài giây connect relay
3. Trong thời gian đó, AI client gửi request → timeout
4. Nếu peer không connect được, tất cả requests timeout vĩnh viễn

### Fix: retry connect socket trước khi fall back

```js
// lib/mcp.js — code mới
const RECONNECT_DELAY = 2000;
const MAX_RECONNECTS  = 15;   // ~30s — đủ cho service restart

function bridge(attempt = 0) {
  if (!existsSync(ipcPath)) {
    if (attempt >= MAX_RECONNECTS) return standalone();
    return setTimeout(() => bridge(attempt + 1), RECONNECT_DELAY);
  }
  // ... connect, pipe stdin↔stdout
  function reconnect() {
    socket.destroy();
    const next = connected ? 0 : attempt + 1;
    if (next >= MAX_RECONNECTS) return standalone();
    setTimeout(() => bridge(next), RECONNECT_DELAY);
  }
}

bridge();
```

**Behaviour mới khi service restart:**
- Socket close → đợi 2 giây → thử kết nối lại
- Retry tối đa 15 lần (~30 giây) — đủ cho `systemctl restart zen.service`
- Nếu socket không xuất hiện sau 30s → mới fall back sang standalone
- Khi reconnect thành công → reset attempt counter

**Đây không phải monkey patch.** Đây là sửa source `lib/mcp.js` thật sự. Có hiệu lực sau lần publish npm tiếp theo.
