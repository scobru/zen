## 1. Triển khai relay ZEN với HTTPS — `peer0.akao.io:8420`

### 1.1 Môi trường

Máy chủ là Orange Pi chạy tại nhà với:
- IPv4 động `42.115.236.139` qua proxy/CDN (trả về 503, không reach được từ ngoài)
- IPv6 động `2405:4802:1bbd:cc0:...` — địa chỉ thực, thay đổi theo modem
- Domain `peer0.akao.io` quản lý bởi GoDaddy (`ns15/ns16.domaincontrol.com`)
- DDNS được xử lý bởi `akaoio/access` chạy ngầm — cập nhật DNS khi IP thay đổi

### 1.2 Vấn đề SSL — tại sao standalone mode thất bại

Let's Encrypt standalone challenge yêu cầu port 80 reachable từ internet. Không thể dùng vì:
- nginx đang chiếm port 80 (cài mặc định, không dùng) — đã gỡ bằng `apt purge nginx`
- Sau khi gỡ nginx, vẫn không dùng standalone được vì IPv4 đi qua proxy → HTTP 401 từ LE
- IPv6 là địa chỉ thực nhưng dynamic → DNS validation là con đường duy nhất

### 1.3 Giải pháp — acme.sh với GoDaddy DNS API

```bash
export GD_Key="..."
export GD_Secret="..."
~/.acme.sh/acme.sh --issue --dns dns_gd -d peer0.akao.io
~/.acme.sh/acme.sh --install-cert -d peer0.akao.io \
  --key-file ~/key.pem \
  --fullchain-file ~/cert.pem \
  --reloadcmd "sudo systemctl restart relay"
```

acme.sh lưu `GD_Key`/`GD_Secret` vào `~/.acme.sh/account.conf` sau lần đầu. Cert ECC, hết hạn ~2026-06-27, tự renew qua cron của acme.sh.

### 1.4 Bug trong `script/server.js` — `ZEN is a class`

Relay khởi động nhưng crash ngay với:

```
TypeError: Class constructor ZEN cannot be invoked without 'new'
```

Nguyên nhân: dòng 222 thiếu `new`:

```js
// SAI
zen = ZEN({ web: opt.server.listen(opt.port), peers: opt.peers })

// ĐÚNG
zen = new ZEN({ web: opt.server.listen(opt.port), peers: opt.peers })
```

`ZEN` là `class` — không thể gọi như function. Fix đơn giản nhưng gây downtime hoàn toàn.

## 2. Rewrite `script/install.sh` — loại bỏ loạt bugs tích lũy

Script cũ có nhiều lỗi do thiết kế tệ:

| Lỗi | Nguyên nhân | Fix |
|-----|-------------|-----|
| `cd: No such file or directory` | `execute()` dùng `timeout 300 "$cmd"` — không thể wrap shell built-in | Bỏ `timeout`, dùng `git -C $dir` và `npm --prefix` |
| `confirm: command not found` | `confirm()` được gọi nhưng chưa định nghĩa | Thêm hàm `confirm()` |
| `git checkout master` fail | Branch đã đổi thành `main` | `VERSION="main"` |
| `ulimit: No such file or directory` | `ulimit` là shell built-in, không thể chạy qua `timeout` | Gọi trực tiếp, không wrap |
| Node.js LTS thay vì 24 | Hardcode NodeSource LTS | `NODE_MAJOR=24` |
| "GUN" khắp nơi | Copy từ upstream GUN chưa rename | Rename toàn bộ GUN → ZEN |

Script mới dùng pattern `run()` trực tiếp:

```bash
run() {
    log_info "Running: $*"
    "$@"   # gọi trực tiếp, không qua timeout
}
```

### 2.1 Thêm `script/update.sh` — tự cập nhật production

## 5. `akaoio/access` — fix race condition khi mất điện

### 5.1 Vấn đề

Ông già tắt điện khi ra ngoài → modem và Orange Pi đều restart. Race condition:
- **Máy boot sau modem**: IP đã được gán TRƯỚC khi `daemon.sh` khởi động → `ip monitor addr` không fire event → DNS không được sync
- **Máy boot trước modem**: OK — `ip monitor addr` sẽ bắt event khi IP xuất hiện

### 5.2 Bug thứ hai — 120s debounce sai thứ tự trong `sync.sh`

```sh
# BUG: check thời gian TRƯỚC KHI check IP thay đổi
if [ "$time_diff" -lt 120 ]; then
    return 0  # skip dù IP đã thay đổi!
fi
```

`flock` đã chống concurrent, `last_ipv4`/`last_ipv6` files đủ idempotency → debounce 120s là redundant và harmful.

### 5.3 Fix

**`daemon.sh`** — thêm `wait_for_network()` và initial sync trước khi vào `ip monitor addr`:

```sh
wait_for_network() {
    attempts=0
    while [ "$attempts" -lt 60 ]; do          # max 2 phút
        ip route show default 2>/dev/null | grep -q "default" && return 0
        sleep 2
        attempts=$((attempts + 1))
    done
    return 1
}

if wait_for_network; then
    "$BIN" sync && log_info "Initial sync done"
fi

# Luôn vào loop này bất kể wait_for_network pass hay fail
while true; do
    ip monitor addr | ...
```

**`module/sync.sh`** — xóa hoàn toàn block debounce 120s.

### 5.4 Resilience layers sau fix

```
Boot → wait_for_network (max 2 phút)
         ├─ network sớm  → initial sync → ip monitor addr canh tiếp
         └─ network muộn → ip monitor addr canh → sync khi IP xuất hiện
              └─ daemon crash → systemd Restart=always → restart sau 5s
                   └─ vẫn miss → access-sync.timer chạy mỗi 5 phút
```
