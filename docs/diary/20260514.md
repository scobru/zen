## 1. ZEN.recover — Prehash mode (03:43)

**Commit:** `3a4a0a8` — `fix: extend ZEN.recover with prehash mode, remove secp256k1 drift from evm.js`

### Vấn đề phát hiện

Trong `lib/chains/evm.js`, hàm `recoverTypedDataAddress()` đang tự implement lại toàn bộ secp256k1 recovery math từ đầu — ~80 dòng bao gồm `_SECP_P`, `_SECP_N`, `_SECP_G`, `_mod`, `_modPow`, `_modInv`, `_pointAdd`, `_pointMultiply`, `_liftX`, `_recoverPubPoint`. Đây là vi phạm nghiêm trọng triết lý kiến trúc: ZEN đã có WASM-accelerated secp256k1 trong `src/curves/`. Không có lý do gì để re-implement affine math ở lớp adapter.

### Giải pháp

Mở rộng `ZEN.recover` để hỗ trợ **prehash mode** — mirrors `ZEN.sign(data, pair, null, { prehash: true })`:

```js
// Trước: evm.js tự tính recovery
const point = _recoverPubPoint(hash, r, s, v); // ~80 dòng affine math

// Sau: delegate hoàn toàn cho ZEN primitive
const addr = await ZEN.recover(rawSig, { prehash: true, hash: digest, format: 'evm' });
```

**API mới của ZEN.recover:**
- `data`: raw sig `{ r, s, v }` object hoặc 65-byte `0x...` hex string
- `opt.prehash: true` — bật prehash mode
- `opt.hash`: pre-hashed 32-byte digest (Uint8Array hoặc hex string)
- `opt.format: 'evm'` → trả về checksummed EVM address; mặc định → ZEN base62 pub key

**Hiệu năng:** WASM Jacobian secp256k1 (~39× nhanh hơn affine JS thuần).

**Tests:** 5 test cases mới trong `test/zen/crypto.js` kiểm tra tất cả input formats.

## 2. Wallet caching + ZEN.TextEncoder/Decoder (04:25)

**Commit:** `54192ce` — `refactor(chains/evm): cache ZEN.pair in Wallet, use ZEN.TextEncoder/Decoder`

### Vấn đề

`Wallet._init()` trước đây gọi `ZEN.pair(format:'evm')` mỗi lần cần sign transaction — mỗi lần là một secp256k1 scalar multiply. Với DeFi workflows (swap + approve + nhiều tx liên tiếp), đây là waste không cần thiết.

### Giải pháp

```js
// Wallet._init() — gọi một lần, cache kết quả
async _init() {
  this._pair = await ZEN.pair({ seed: this._priv, curve: 'secp256k1', format: 'evm' });
  this.address = this._pair.address;
}

// signTransaction() — dùng cache
async signTransaction(tx) {
  const pair = this._pair || await ZEN.pair({ seed: this._priv, curve: 'secp256k1', format: 'evm' });
  // ...
}
```

`Wallet.connect()` giờ carry over `_pair` (không chỉ `address`) để chain tiếp tục dùng được.

Ngoài ra, `strToBytes()` và ABI decode chuyển sang dùng `ZEN.TextEncoder` / `ZEN.TextDecoder` — nhất quán với pattern `ZEN.Buffer` / `ZEN.random`. `src/index.js` expose thêm hai utility này từ shim.

## 3. AXE → src/ — Di chuyển vào core bundle (12:22 — 12:25)

**Commits:** `ec77632`, `048301d`

### Vấn đề kiến trúc (từ issue #28)

AXE (Automatic eXchange of peers) là thành phần quan trọng không thể tách rời khỏi ZEN — nhưng trước đây có asymmetry giữa hai môi trường:

| Môi trường | Cách load AXE |
|---|---|
| Node.js | Tự động — `index.js → lib/server.js → lib/axe.js` |
| Browser | Thủ công — `import "./lib/axe.min.js"` (user phải tự làm) |

User phải biết import thêm file riêng là một friction không cần thiết.

### Thách thức kỹ thuật: Circular dependency

`lib/axe.js` line 1: `import ZEN from "../zen.js"` — nếu move vào `src/` và thêm `import "./axe.js"` vào `src/index.js`, ES6 static imports bị hoist → axe.js execute trước khi ZEN class được define.

**Giải pháp:** Factory pattern:

```js
// src/axe.js — sau refactor
export default function initAXE(ZEN) {
  // Tất cả code AXE, dùng ZEN parameter thay vì import
}

// src/index.js — cuối file, sau khi ZEN đã define
import initAXE from "./axe.js";
initAXE(ZEN);
```

### Vấn đề thứ hai: Bundler không handle `export function`

Builder tại `lib/builder/zen.js` chỉ handle các patterns:
- `export { a, b }` ✓
- `export default function name(` ✓
- `export default expr;` ✓

Nhưng KHÔNG handle `export function mkpat(d)` (named export của function declaration). `src/scan.js` dùng pattern này → bundler include raw `export` keyword vào zen.js → terser fail khi minify.

**Giải pháp:** Convert `scan.js` sang builder-compatible style:
```js
// Từ: export function mkpat(d) { ... }
// Thành: function mkpat(d) { ... }
// Và thêm ở cuối: export { mkpat, bdom, candidateHosts, scan, scanip6, scanbg };
```

### Vấn đề thứ ba: DHT test fail sau khi AXE vào core

Khi AXE nằm trong `zen.js`, tất cả tests import zen.js → AXE opt handler chạy trong mọi test. Test DHT (`test/dht/kbucket.js`) comment rõ "skipping browser/axe code" — nhưng giờ không skip được nữa. AXE trong Node.js path ghi đè `mesh.way` → DHT message routing bị ảnh hưởng → `fn_r` không đến được → timeout 3000ms.

**Điều tra:** Confirm `mesh.way` bị set bởi AXE Node.js path, trong khi test dùng `WebSocket: false`.

**Fix:** Thêm guard ở đầu `start()` trong AXE — trước khi bất kỳ mutation nào xảy ra:

```js
function start(root) {
  if (root.axe) { return; }
  var opt = root.opt, peers = opt.peers;
  if (false === opt.axe) { return; }
  if (false === opt.WebSocket) {
    return; // in-process / test mode — no real network, skip AXE entirely
  }
  // ...
}
```

`WebSocket: false` có nghĩa là "không có real network" — semantic hoàn toàn đúng để skip AXE.

### Quyết định về backward compat

Ban đầu giữ `lib/axe.js` và `lib/axe.min.js` như "shim" (empty comment file). User nhắc: **system chưa launch, không cần backward compat**. Xóa sạch hoàn toàn, commit riêng `048301d`.

### Kết quả

- Browser: không cần `import axe.min.js` nữa — AXE đã trong `zen.js`
- Node.js: không thay đổi behavior với server thực (có WebSocket)
- Tests: không bị ảnh hưởng (`WebSocket: false` → AXE skip)
- `lib/minify.js`: 92 files (từ 93) — `lib/axe.js` đã biến mất sạch

### `npm start` → `script/server.js`

Cùng trong commit `ec77632`, `package.json` được cập nhật:
```json
"start": "node script/server.js"  // từ "node relay.js"
```

`script/server.js` đã zero-config safe từ trước (domain null → auto-latch, TLS optional, identity failure caught, `disc()` async non-blocking).

## Bài học kiến trúc hôm nay

**1. "Di chuyển" khác với "copy"**
Khi move AXE vào `src/`, phải hiểu tại sao `lib/axe.js` tồn tại ở `lib/` chứ không phải `src/` — circular dep. Giải quyết bằng factory pattern thay vì brute force move.

**2. Bundler là contract**
Builder `lib/builder/zen.js` có contract riêng về export syntax. `export function name()` không được hỗ trợ. Phải hiểu contract của tool trước khi di chuyển code.

**3. `WebSocket: false` là signal mạnh**
Trong ZEN ecosystem, `WebSocket: false` nghĩa là "không có real network". AXE (peer discovery) không nên chạy khi không có network. Guard này có semantic rõ ràng và được tái sử dụng ở nhiều chỗ (websocket.js cũng check pattern này).

**4. Debug logs phải có vòng đời**
`[BYE]` với `new Error().stack` capture — hữu ích trong 30 phút debug, nhưng nếu không xóa sẽ là overhead vĩnh viễn. Nguyên tắc: debug log nên được commit với comment `// TODO: remove` hoặc wrap trong `if (process.env.ZEN_DEBUG)`.

**5. "Không cần backward compat" là quyết định quan trọng**
System chưa launch → không có user nào depend vào `lib/axe.js`. Việc giữ shim chỉ tạo ra technical debt. Move fast, xóa sạch.
