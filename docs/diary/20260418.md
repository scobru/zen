## Dọn nhà — dead code, builder, và câu hỏi "cái này có dùng không?"

Hôm nay không có feature mới. Hôm nay là ngày dọn dẹp — loại bỏ những thứ không còn tồn tại, di chuyển những thứ còn lại về đúng chỗ, và đảm bảo tài liệu phản ánh code thật chứ không phải code cũ.

Tổng kết một dòng: `~@` alias system hoàn toàn biến mất, `lib/builder/` có vị trí chính thức, và `examples/contact/index.html` không còn gọi API không tồn tại nữa.

## Buổi sáng: Dead code — `~@` alias system

### Bối cảnh

ZEN là fork của GunDB. GunDB có một hệ thống tài khoản gắn với `user.create()` / `user.auth()` — người dùng đăng ký bằng alias (username), alias được lưu vào namespace `~@<alias>` trên graph. Đây là lớp trừu tượng để người dùng tìm nhau qua tên thay vì public key.

Hệ thống đó đã bị xóa khỏi fork này. `zen.user()` không tồn tại. Nhưng trong `src/security.js` vẫn còn hai hàm `check.alias` và `check.pubs` — viết ra để bảo vệ namespace `~@`, một namespace không ai còn ghi vào. Pipeline vẫn route `~@` soul qua các handler đó.

Đây là định nghĩa kinh điển của dead code: code chạy được, không báo lỗi, nhưng không có caller thật, không có data thật, không có tác dụng thật.

### Quyết định

Xóa sạch:
- `check.alias` — hàm kiểm tra alias index
- `check.pubs` — hàm kiểm tra pub list dưới `~@`
- `check.pipe.alias`, `check.pipe.pubs` — các entry trong pipeline
- Tất cả routing branch dành cho soul `~@...`

Pipeline selector sau khi dọn:

| Soul | Handler |
|------|---------|
| `~` hoặc `~/` | `check.shard` |
| match `settings.pub(soul)` | `check.pub` |
| có `#` trong soul | `check.hash` |
| còn lại | `check.any` |

Sau khi xóa: `npm run buildZEN`, test vẫn `171 passing`. Dead code thật sự — không có gì phụ thuộc vào nó.

### Tài liệu theo sau code

Xóa code xong, cần cập nhật tài liệu. `ch04-authenticated-data.md` vừa viết xong tuần trước còn có section "Alias index" giải thích `~@`. Xóa section đó. Bảng namespace trong 4.2 và bảng pipeline trong 4.13 bỏ hết row liên quan đến `~@`.

Cũng cập nhật `test/panic/who.js` — file test dùng `zen.user()`, `user.create()`, `user.auth()`, `~@alice` lookup. Rewrite toàn bộ: tạo identity bằng `ZEN.pair()`, lưu encrypted pair vào `localStorage`, discovery qua `zen.get("discovery").get("alice")` thay vì `~@alice`.

## Buổi chiều: `examples/contact/index.html`

File này là ví dụ contact/social app từ thời GunDB. Script block của nó dùng toàn bộ API đã bị xóa:

- `var user = zen.user()` — không tồn tại
- `zen.on('auth', ...)` — event không còn fire
- `zen.on('secure', ...)` — bao gồm dòng `if('~@'+key === ...)` — rõ ràng là `~@` dead code
- `user.create()`, `user.auth()` — không tồn tại
- `user.is`, `user._.pub` — không tồn tại
- Trang `converse` dùng biến `sign` không được khai báo ở đâu — luôn throw ReferenceError từ đầu

Thêm vào đó, khi đọc kỹ trang `converse`, nó còn dùng `priv` (biến private key raw) — pattern cũ của GunDB không còn tồn tại trong ZEN.

### Rewrite

Thay thế toàn bộ script block:

**Sign up:** `ZEN.pair()` → `ZEN.encrypt(JSON.stringify(pair), passphrase)` → lưu vào `localStorage`. Ghi profile lên `~<pub>` với `{ authenticator: pair }`. Ghi `{ pub }` lên `app.get('users').get(alias)` cho discovery.

**Sign in:** đọc encrypted pair từ `localStorage` → `ZEN.decrypt(enc, passphrase)` → parse → set `c.me`.

**`#say` submit:** ghi vào `zen.get('~' + c.me.pub).get('who').get('said').set(...)` với `{ authenticator: c.me.pair }`.

**Route guards:** `if(!c.me){ return as.route('sign'); }` thay cho `if(!user.is)`.

**Trang `converse`:** xóa. Không migrate — logic messaging qua graph cần thiết kế lại từ đầu, không phải patch trên foundation đã hỏng.

Kết quả: một script 150 dòng thay cho 170 dòng dead code, không có undefined variable, không có API không tồn tại.

## Buổi chiều muộn: Builder reorganization

Một task nhỏ nhưng rõ ràng: `lib/build-zen.js`, `lib/build-pen.js`, `lib/build-crypto.js` nằm lẫn với hàng trăm storage adapter và utility module trong `lib/`. Thư mục `lib/builder/` đã tồn tại nhưng trống.

Di chuyển ba file vào `lib/builder/` với tên ngắn: `zen.js`, `pen.js`, `crypto.js`.

Sửa `rootDir` trong mỗi file từ `".."`  thành `"../.."` — vì bây giờ chúng nằm sâu hơn một cấp.

Cập nhật `package.json`:
```json
"buildZEN":    "npm run buildPEN && npm run buildCrypto && node lib/builder/zen.js && ...",
"buildPEN":    "node lib/builder/pen.js",
"buildCrypto": "node lib/builder/crypto.js"
```

Xóa sáu file cũ (`build-*.js` và `build-*.min.js`). Chạy `npm run buildZEN` — build thành công, `crypto.wasm built: 66138 bytes`.

## Cuối ngày: Câu hỏi nhỏ, trả lời thật

Cuối ngày có hai câu hỏi nhỏ nhưng đúng lúc:

**"sha256.zig và hmac_sha256.zig có phải tech drift không?"**

Không. Cả hai đều được `crypto.zig` import và dùng. `sha256.zig` export trực tiếp ra WASM và là primitive của HMAC. `hmac_sha256.zig` export HMAC ra WASM và là thành phần cốt lõi của RFC 6979 trong `curves/utils.zig` — mỗi lần ký ECDSA đều chạy qua nó nhiều lần.

**"Sao `ch04-authenticated-data.md` bị git ignore?"**

`.gitignore` có rule `*data*` để ignore `radata/`, `radatatest/`. Rule đó vô tình match `ch04-authenticated-**data**...md`. Thêm `!docs/**` exception vào ngay sau rule đó. File được git nhận ra ngay lập tức.

## Buổi tối: Dọn tên biến — xóa `__` prefix

### Bối cảnh

ZEN là fork của GunDB. GunDB dùng pattern `import __GUN from '../gun.js'` — double underscore prefix để tránh đụng tên với `var GUN` khai báo sau đó trong wrapper block. Khi fork thành ZEN, pattern đó kéo theo.

Trong session trước, một global find-and-replace `__zen → ZEN` và `__ZEN → ZEN` đã làm vỡ toàn bộ test suite vì cả import lẫn `var ZEN` đều thành `ZEN` — ES module báo `Identifier 'ZEN' has already been declared`.

Session này sửa hậu quả đó, rồi đi xa hơn: xóa `__` prefix khỏi tất cả user-defined names trong codebase. Lý do: quy ước `__` prefix là C++/CPP convention dành cho compiler internals. Trong JS, nó không có nghĩa kỹ thuật — chỉ là visual noise.

### Những gì thay đổi

Tất cả test files trong `test/`, bao gồm `test/rad/`, `test/panic/`, `test/old/`, `test/https/`, `test/bug/`:

| Tên cũ | Tên mới |
|--------|---------|
| `__ZEN` | `zenbase` |
| `__fs` | `fs` |
| `__fsrm` | `fsrm` |
| `__expect` | `xpect` |
| `__serve` | `serve` |
| `__pen` | `penmod` |
| `__defaultExport` | `defaultexport` |
| `__dirnameOf` | `dirnameOf` |
| `__child_process` | `childproc` |
| `__https` | `nodehttps` |
| `__path` | `nodepath` |
| `__index` | `zenapp` |
| `__radisk` | `radisk` |
| `__rfs` | `rfs` |
| `__radix` | `radix` |
| `__express` | `express` |
| ... | ... |

`__filename` và `__dirname` được giữ nguyên — đây là Node.js CJS module globals, không phải user-defined.

Builder template (`lib/builder/zen.js` và `zen.min.js`) cũng được cập nhật: các tên internal `__mods`, `__def`, `__req`, `__exp`, `__zen` trong code generated ra `zen.js` đổi thành `mods`, `defmod`, `reqmod`, `exp`, `zenmod`.

## Nhìn lại

Hôm nay không có dòng code nào tạo ra capability mới. Mọi thứ đều là xóa, di chuyển, sửa lại, làm đúng những gì đã có.

Nhưng cảm giác cuối ngày khác với đầu ngày. Repo bây giờ không có dead code lộ thiên, không có example gọi API không tồn tại, không có builder script nằm lẫn giữa adapter library, không có doc file bị ignore lặng lẽ, không có `__` noise trong tên biến, không có output test rời rạc.

Đây là loại công việc mà nếu không làm thì không ai thấy thiếu — cho đến khi ai đó vào repo lần đầu và hỏi "cái này dùng để làm gì?", rồi phát hiện ra câu trả lời là "không làm gì cả".
