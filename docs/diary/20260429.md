## 1. Scan vô hạn — bandwidth ngốn không dừng

### Vấn đề

`scheduleScan()` chạy mỗi 10 phút mãi mãi dù không tìm thấy peer nào. Mỗi cycle gửi ~200 HTTP probe. Với 10 peer cùng làm điều này → hàng nghìn request vô nghĩa/giờ.

### Fix

**`scan()` — dừng sớm khi đủ peers** (`maxFound = MFND = 10`):

```js
// lib/scan.js
const MFND = 10   // mfnd = max found per cycle
// Worker task sets stp = true once found.length >= mfnd
// Remaining queued tasks check: if (stp) return null
```

**`schd()` — exponential backoff khi cycle trống**:

```
Empty cycle 1  →  next scan sau 20 phút  (siv = siv * 2)
Empty cycle 2  →  sau 40 phút
...
Cap            →  2 giờ (MSIV)
Discovery      →  reset về 10 phút ngay lập tức
Peer disconnect (bye) →  reset và scan lại ngay
```

Backoff theo dõi bằng cờ `fic` (foundInCycle). Đặt `fic = false` đầu mỗi `schd()` tick; `adp()` đặt `fic = true` khi thêm peer thành công. Sau 2 phút kiểm tra: nếu `!fic` → tăng `siv`.

## 2. Full-mesh — tất cả kết nối tất cả, mạng tự chết

### Vấn đề

AXE MOB chỉ giới hạn **inbound** — khi `mesh.near >= opt.mob`, AXE redirect peer mới đến nơi khác. Nhưng **outbound** từ scan (`pmsh.hi()`) không có giới hạn. Nếu scan tìm được 100 peers và cố kết nối tất cả → O(n²) links → mạng chết.

### Fix

**`adp()` kiểm tra `r.axe.up` trước khi gọi `pmsh.hi()`**:

```js
const MUPS = 10
// adp = add peer
function adp(url) {
  ...
  const ups = Object.keys(r.axe.up || {}).length
  if (pmsh && ups < MUPS) pmsh.hi({ id: url, url, retry: 9 })
  // PEX broadcast vẫn xảy ra dù ups >= MUPS
}
```

`r.axe.up` là object trong `lib/axe.js` tracking các kết nối upstream đang hoạt động. Không cần thêm counter riêng — nguồn truth duy nhất.

**Hai lớp giới hạn topology**:

| Layer | Cơ chế | File |
|-------|--------|------|
| Inbound | AXE MOB — redirect khi `mesh.near >= opt.mob` | `lib/axe.js` ~491 |
| Outbound | `MUPS = 10` — scan không gọi `hi()` nếu đã đủ | `script/server.js` `adp()` |

## 4. Peer Exchange (PEX)

Khi peer mới kết nối:

```js
// server.js — "hi" event
pmsh.on("hi", peer => {
  // Gửi danh sách peers đang biết
  pmsh.say({ dam: "pex", peers: knownUrls }, peer)
})

// Handler nhận pex
pmsh.hear["pex"] = (msg, _peer) => {
  msg.peers.forEach(url => adp(url))
}
```

Khi tìm thấy peer mới qua scan, broadcast ngay cho tất cả peers đang connected — không đợi đến lần kết nối tiếp theo.

PEX **không dùng public graph** — giao tiếp trực tiếp qua DAM `"pex"` message. An toàn với graph có policy bảo mật.

## 5. Self-discovery (`lib/discover.js`)

Peer cần biết domain/IP của mình để build URL tự giới thiệu. Ưu tiên offline-first:

1. `~/.config/zen/domain` — file config do install.sh hỏi khi setup
2. `ip route get 8.8.8.8` → parse output lấy `src <IP>`
3. `hostname -I` → lấy IP đầu tiên không phải loopback
4. STUN `stun.l.google.com:19302` — RFC 5389 Binding Request → XOR-MAPPED-ADDRESS
5. `api.ipify.org` / `ifconfig.me` — HTTP fallback

Domain được lưu lại vào `DOMF` lần đầu tìm được. Nếu vẫn chưa biết khi server start, `ldom()` trong `server.js` bắt domain từ `Host` header của request đến đầu tiên.

## 6. Cấu trúc scan thông minh

Từ domain `peer1.akao.io`, scanner tự suy ra pattern:

```
mkpat("peer1.akao.io") → { prefix: "peer", index: 1, suffix: ".akao.io", padLen: 0 }
bdom(pat, 0)  → "peer0.akao.io"
bdom(pat, 2)  → "peer2.akao.io"
bdom(pat, 15) → "peer15.akao.io"
```

Với zero-padding (`relay01.host`):
```
mkpat("relay01.host") → { prefix: "relay", index: 1, suffix: ".host", padLen: 2 }
bdom(pat, 5) → "relay05.host"
```

Probe song song (`CONC = 10`), HTTPS trước HTTP, timeout 3s. Dừng sau `MFND = 10` peers.

## 9. Định hướng

- **`script/install.sh --domain`**: Hỏi domain khi setup thay vì để peer tự phát hiện sau — giảm cold-start latency
- **Scan pattern mở rộng**: Khi PEX chia sẻ peers với domain khác hẳn (ví dụ `zen.duke.io.vn`), scan pattern của domain đó cũng nên được thêm vào `spat` để mở rộng mạng
- **IPv6 scan**: Hiện tại chỉ scan domain-based; có thể thêm IPv6 link-local multicast
- **HAM-secure PEX**: PEX hiện không ký — có thể add optional signature để peer verify nguồn thông tin
