## Ngày dọn lib — và những bài test cuối cùng trở thành xanh

Hôm nay là ngày hoàn thành một việc đã kéo dài nhiều session: di chuyển toàn bộ `lib/*.js` từ `gun.js` sang `zen.js`, loại bỏ mọi `globalThis.GUN` / `globalThis.Gun` khỏi hệ thống.

---

## 1. Tại sao phải làm việc này

Khi `zen.js` được xây dựng, `gun.js` vẫn tồn tại song song. Tất cả các adapter trong `lib/` đều import từ `gun.js` và đôi khi truy cập qua `globalThis.Gun` — một pattern nguy hiểm: bất kỳ ai kiểm soát `globalThis` đều có thể inject dữ liệu bẩn vào toàn bộ graph.

Mục tiêu hôm nay: **không còn `globalThis.Gun`, không còn `globalThis.GUN`**. Toàn bộ lib phải dùng `zen.js` trực tiếp qua ESM import.

## 3. Test regressions và cách fix

Sau khi migrate lib xong, test chạy nhưng có 2 nhóm lỗi mới.

### Lỗi 1: `gun._.root` là `undefined`

**Root cause**: `ZEN` class dùng proxy pattern. Các method như `.get()`, `.put()`, `.on()` đều delegate sang `this._graph` (instance `Gun` nội bộ). Nhưng `ZEN` prototype không expose `._` — điều mà hàng chục chỗ trong test/common.js cần để truy cập `gun._.root`, `gun._.graph`, `gun._.stun`.

```js
// nopasstun() trong test/common.js:
var nopasstun = function(done, g){
    g = (g || gun)._.root;  // gun._ là undefined → crash
    ...
};
```

**Fix**: Thêm getter `_` vào wrapper factory trong `test/common.js` và `test/index.js`:

```js
Object.defineProperty(W.prototype, '_', {
    get: function(){ return this._graph._; },
    configurable: true
});
```

Đơn giản. Clean. `this._graph._` là context nội bộ của Gun instance bên trong ZEN, đầy đủ `root`, `graph`, `stun`, `dup`, tất cả.

### Lỗi 2: `Gun.is(gun)` trả về `false`

**Root cause**: `Gun.is` trong gun.js được định nghĩa là:
```js
Gun.is = function($){ return ($ instanceof Gun) || ($ && $._ && ($ === $._.$)) || false }
```

`gun` là `ZEN` instance, không phải `Gun` instance (internal bundled Gun trong zen.js). `instanceof` fail. `$._ && ($ === $._.$)` fail vì `gun._.$` là `gun._graph` (Gun nội bộ), không phải `gun` (ZEN wrapper).

Thêm vào là: `test/rad/rad.js` chạy trước `test/common.js` và set `global.Gun = W` nhưng W lúc đó chưa có fix `W.is`. Nên khi `test/common.js` chạy, `var Gun = root.Gun` lấy W cũ, `Gun.is` vẫn là internal Gun.is.

**Fix**: Override `Gun.is` unconditionally sau dòng `var Gun = root.Gun`:

```js
var Gun = root.Gun;
Gun.is = function($){ return $ instanceof __ZEN; };
```

Và thêm `W.is` vào factory luôn để đảm bảo nhất quán:

```js
W.is = function($){ return $ instanceof __ZEN; };
```

### Lỗi 3: "Check multi instance message passing" timeout

**Root cause**: Test này dùng `Gun.on('opt', ctx)` để bridge messages giữa 2 instances `b` và `d`. Khi instance `b = Gun({file: "bdata"})` tạo một ZEN wrapper, `Gun.on('opt', ...)` fire với `ctx.$` là *inner* Gun instance (`b._graph`), không phải `b` (ZEN wrapper). So sánh `onGun.$ === b` luôn fail.

**Fix**: So sánh thêm `_graph`:

```js
if(onGun.$ === b || onGun.$ === (b && b._graph)) {
    // bridge b → d
} else if(onGun.$ === d || onGun.$ === (d && d._graph)){
    // bridge d → b
}
```

## 5. Tại sao điều này quan trọng

Trước: `lib/` phụ thuộc vào `globalThis.Gun` — ai kiểm soát `globalThis` có thể inject Radisk/Radix giả, hoặc swap toàn bộ Gun instance. Với ứng dụng local-first P2P offline, đây là surface attack thực.

Sau: `lib/` không còn nhìn vào `globalThis` cho identity của Gun. Mọi thứ đến từ ESM import tĩnh. `zen.js` là entry point duy nhất. Không có leak.

---

## 6. Nhìn lại

Migration này không phải refactor vì muốn đẹp. Nó là bước cần thiết để ZEN thực sự là ZEN — không phải GUN với tên mới.

Một ký tự thay đổi trong import: `gun.js` → `zen.js`. Nhưng sau đó là một loạt domino cần giải quyết: script viết, script bug, script fix, manual patches, test regressions, test infrastructure không hiểu ZEN instance là gì. Mỗi bước đều có lý do rõ ràng.
